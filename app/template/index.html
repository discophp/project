This Page (The index)

This is your index, and the content is coming from app/template/index.html

Your seeing this because the route / has been resolved by a router in app/index.php to the root controller App\controller\Root and the method getIndex() was called and loaded this template into the View.

The View

The View App\view\Standard is being used and by default is using the template app/template/_default.html as its baseTemplate to format View content. The default template is extending the app/template/_layout.html which provides the generic HTML structure and resource handling.

The methods header() & footer() are defined on App\view\Standard which are loading the templates app/template/header.html & app/template/footer.html into the View and are output in the default template.

The View in its __construct() method is also loading some resources which are helping style this page, primarly Materialize CSS.

What Next

Check out the default user functionality that is built into Disco by checking out the files in app/router/user.router.php, app/controller/User.php, app/controller/UserPublic.php, app/service/User.php, app/template/user/, app/data_model/User.php.

Notice : To use the default user functionality that comes with disco you will need to create the MySQL table dependencies. To do so first make sure your database configuration settings are properly defined in app/config/config.php, then execute the command php public/index.php db-restore 'app/db' 'user_structure.sql' to create the SQL tables. To finalize the configuration for using the built in user functionality configure your email settings in app/config/email.php to enable verifying accounts and password resets.
